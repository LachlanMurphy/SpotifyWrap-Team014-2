Three tests:

1:A user must first register, then login successfully before accessing the home page
    Mandatory steps:
        - User must register
            - Must submit email and password
        - User must login
            - Must submit an email and corresponding password
        - If user successfully completed previous steps, send to home page with logged in session
        - If user does not succsffuly complete previous steps, send to where the error is (login or registration page)

    Postive test:
        - Must return a success status when logged in and registered
    Negitive test:
        - A non registered user must not be able to log in
        - A non logged in session must not be able to access the home page

2:A user that is currently logged in should be able to access their profile from the home page
    - To access their profile, a user can click on their profile name to go to their profile page
    - In the profile page the user can edit their username or password with the following fields:
        - current username
        - new username
        - current password
        - new password
    Positive test:
        - Must return a success status and user information when finished
    Negitive test:
        - A non logged in session must not be able to access the profile page
    

3:A logged in user should be able to insert three songs that they like and get similar songs that they may like
    - Required fields for the search page:
        - Song #1
        - Song #2
        - Song #3
    - After clicking submit the user should get a list of songs that the spotify API reccomends that are similar
      to the three songs they submited.
    Positive tests:
        - A search request should take 3 inputs and return a list of songs
    Negitive tests:
        - A sesssion should not be able to access the search page if they are not logged in
        - A search must not process unless 3 input fields are given by the user
